{{ page }}. Part-2 Session Key Exchange

  1. Once the validation is successful then the web-browser extracts public key from rothchild.crt which will be same as rothchild.pub
  2. Generates a sync encryption key like AES key, encrypt the AES key using rothchild.pub and send this to web-server.
  3. The web-server receives the encrypted AES key and decrypts it using it's private key rothchild.pvt
  4. From now on every communication between the web-server and the web-bowser will be encrypted using this AES key.
